# Цель работы

Знакомство с облачными сервисами.
Понимание уровней абстракции над инфраструктурой в облаке.
Формирование понимания типов потребления сервисов в сервисной-модели. 


------------------------------------------------------------------------

# **Исходные данные**

В рамках лабораторной работы использовался файл биллинга AWS в формате CSV, содержащий информацию о потреблении облачных сервисов. Каждая запись файла описывает отдельный элемент использования сервиса и включает следующие поля:

1.Product Code — идентификатор сервиса AWS

2.Usage Type — характеристика типа потребления ресурса

3.lineItem/Operation — выполняемая операция

4.lineItem/LineItemDescription — текстовое описание строки биллинга

------------------------------------------------------------------------

# **Ход работы**

<img width="326" height="333" alt="image" src="https://github.com/user-attachments/assets/f410ca93-294c-435d-b5c3-e9ce0366199a" />


1. Был получен файл биллинговых данных AWS в формате CSV. Файл импортирован в табличный редактор Google Sheets.

2. После импорта была выполнена проверка корректности структуры таблицы и наличия необходимых полей, используемых для анализа потребления сервисов: Product Code, Usage Type, lineItem/Operation, lineItem/LineItemDescription.

3. В таблице были определены пять классификационных колонок сервисной модели: IT Tower, Service Family, Service Type, Service Sub Type, Service Usage Type. Эти колонки предназначены для описания иерархии и типа потребления облачных сервисов.

4. Для каждой строки биллинга был определён используемый сервис AWS на основе значения поля Product Code. Далее сервис был отнесён к соответствующей функциональной группе IT Tower исходя из его назначения.

5. Для каждого сервиса была определена модель предоставления Service Type. Все сервисы были классифицированы как PaaS, поскольку управление инфраструктурой и платформой осуществляется провайдером AWS.

6. Для уточнения назначения сервиса была заполнена колонка Service Sub Type, отражающая конкретную функцию сервиса внутри выбранной группы.

7. Тип потребления ресурса Service Usage Type определялся на основе анализа значений в колонках Usage Type и lineItem/LineItemDescription. В зависимости от характера использования строки биллинга были отнесены к категориям Compute, Storage, Data Transfer, Requests, Messaging, Backup, Pages или Tax.

8. Заполнение таблицы выполнялось группами по одинаковым значениям Product Code. Для таких строк значения колонок IT Tower, Service Family, Service Type и Service Sub Type оставались неизменными, а Service Usage Type менялся для каждой строки отдельно.


Итоговый файл с заполненной сервисной моделью был сохранён и загружен.

# **Выводы**

В ходе выполнения лабораторной работы были изучены основные принципы классификации облачных сервисов Amazon Web Services и подходы к анализу их потребления на основе биллинговых данных.

Была сформирована сервисная модель, позволяющая распределить использование облачных ресурсов по иерархии — от функциональных групп  до конкретных типов потребления. Это дало возможность наглядно представить структуру облачных сервисов и понять, за какие именно действия и ресурсы происходит начисление стоимости.

В процессе работы получены практические навыки сопоставления строк биллинга с функциональным назначением сервисов, а также классификации сервисов по модели предоставления PaaS. Проведённая классификация позволяет использовать биллинговые данные не только для учёта затрат, но и для анализа эффективности использования облачной инфраструктуры.

Таким образом, поставленная цель лабораторной работы была достигнута.
